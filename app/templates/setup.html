SSH Key & 1Password

Your SSH key lives in 1Password and gets served through its SSH agent. If you already have keys loaded (Step 1 shows green), skip ahead.

1Password v8 Required

Make sure you're running 1Password version 8 — older versions don't have the SSH agent or CLI features we need.

Create an SSH key in 1Password

  1. Open 1Password → click + New Item → SSH Key
  2. Choose Ed25519 (recommended) and give it a name like Work Laptop
  3. Save it — 1Password now manages this key for you

Enable SSH Agent & CLI

  1. In 1Password → Settings → Developer
  2. Turn on "Use the SSH agent"
  3. Turn on "Enable CLI" (also called "Integrate with 1Password CLI")

Configure your SSH config

Add this to ~/.ssh/config so all SSH tools use the 1Password agent:

~/.ssh/config 
Host *
    IdentityAgent "~/Library/Group Containers/2BUA8C4S2C.com.1password/t/agent.sock"

LDAP Password (SSP)

When you first connect via sshv, it will prompt for your LDAP credentials. You need to set these up first.

Set up your LDAP password

  1. Go to the SSP Portal: ssp-ext.voltagepark.net
  2. Sign in using your Voltage Park Google identity
  3. Enter your username (the part before @voltagepark.com) and your Voltage Park email address
  4. Click "Send" to receive a password reset link
  5. Follow the link in your email to set your LDAP password
  6. (Optional) Add your public SSH key in the >_ SSH Key section

What's LDAP?

LDAP is the authentication system for SSH access to servers. When sshv prompts for credentials, use your Voltage Park username (e.g., jmannix) and the password you set in SSP.

Install & Configure sshv

sshv is our SSH wrapper — it handles Vault authentication, jump hosts, and key management. Without it, nothing else works.

Install sshv

It's a private binary, so you need GitHub access (Step 3) before you can download it:

  1. Go to github.com/voltagepark/sshv/releases
  2. Download the latest release for your platform (darwin_arm64 for Apple Silicon Macs)
  3. Install it:
terminal 
sudo mv ~/Downloads/sshv_*_darwin_arm64 /usr/local/bin/sshv
sudo chmod +x /usr/local/bin/sshv
sudo xattr -dr com.apple.quarantine /usr/local/bin/sshv

The xattr command removes the macOS quarantine flag so it can run.

Configure sshv with the portal script

Run the sshv portal script to set up your token. This will prompt for your LDAP credentials (from Step 5):

terminal 
~/Github/Support-Tooling/scripts/sshv/connect-to-sshv-portal.sh

The script will guide you through authentication. Once complete, your token is stored and sshv will work automatically.

Create an alias (optional but handy)

terminal 
~/Github/Support-Tooling/scripts/sshv/connect-to-sshv-portal.sh --create-alias

This creates a connecttosshvportal alias for quick access.

VOLT API Key

Node Toolkit uses the VOLT API to discover nodes by hostname. You'll need an API key to use discovery features.

Generate your API key

  1. Go to volt.lightning.ai/settings
  2. Click "Generate API Key"
  3. Copy the key — you'll only see it once

Storage (automatic)

Node Toolkit handles this for you — on first run it will:

  1. Check for 1Password CLI (install if missing)
  2. Prompt you to paste your API key
  3. Store it securely in your 1Password vault

The key is stored in 1Password under VOLT API Token and retrieved automatically on future runs.

What's VOLT?

VOLT is the infrastructure dashboard — it's the single source of truth for node info (hostnames, IPs, iDRAC addresses, GPU types, customer assignments). When you use Node Toolkit's discovery feature, it queries VOLT to find nodes.
